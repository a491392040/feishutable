# 多维表格合并插件 - 功能说明文档

## 版本信息

- 前端版本：v1.4.1
- 后端版本：latest

---

## 一、功能概览

| 功能 | 说明 |
|------|------|
| 多源表合并 | 支持选择多个源表，合并到同一个目标表 |
| 字段映射 | 源表字段 → 目标表字段映射，支持默认值 |
| 去重 | 按指定字段去重，支持跳过/覆盖两种策略 |
| 字段拆分 | 将字段值按分隔符拆分为多条记录 |
| 父子记录关联 | 自动识别并保持父子记录的关联关系 |

---

## 二、操作流程

### 2.1 基本流程

```
步骤1: 选择源表（支持多选）
    ↓
步骤2: 选择目标表
    ↓
步骤3: 配置字段映射
    ↓
步骤4: 配置去重（可选）
    ↓
步骤5: 配置拆分（可选）
    ↓
步骤6: 预览 & 执行合并
```

### 2.2 源表选择模式

**模式一：选择多个源表**
- 从当前多维表格中选择多个表作为源表
- 所有源表的记录将合并到目标表

**模式二：输入 Base Token + Table ID**
- 适用于跨 Base 合并场景
- 需要输入目标 Base 的 PersonalBaseToken 和 Table ID

## 三、字段映射

### 3.1 映射配置

| 配置项 | 说明 |
|--------|------|
| 源字段 | 源表中的字段 |
| 目标字段 | 目标表中的字段 |
| 默认值 | 当源字段值为空时使用（可选） |

### 3.2 映射规则

1. **字段类型兼容**：源字段和目标字段类型应兼容
2. **关联字段处理**：关联字段会被自动识别并特殊处理
3. **默认值生效条件**：源字段值为空（null/undefined/空字符串）时使用默认值

### 3.3 业务流程

```
源记录字段值 → 检查是否为空
    ↓ 是空值
检查是否有默认值
    ↓ 有默认值
使用默认值写入目标字段
    ↓ 无默认值
不写入该字段（目标字段保持原值或为空）
```

## 四、去重功能

### 4.1 配置项

| 配置项 | 说明 |
|--------|------|
| 启用去重 | 是否开启去重功能 |
| 去重字段 | 用于判断重复的字段（可多选） |
| 去重策略 | `跳过` 或 `覆盖` |

### 4.2 去重策略详解

#### 策略一：跳过（skip）

```
源记录 → 计算去重 Key → 检查目标表是否存在相同 Key
    ↓ 存在
跳过该记录，不写入
    ↓ 不存在
写入目标表
```

**适用场景**：保留目标表已有数据，只追加新数据

#### 策略二：覆盖（overwrite）

```
源记录 → 计算去重 Key → 检查目标表是否存在相同 Key
    ↓ 存在
1. 删除目标表中该记录
2. 写入源记录
    ↓ 不存在
写入目标表
```

**适用场景**：用源表数据更新目标表中的重复数据

### 4.3 去重 Key 计算规则

```
Key = 字段值1 ||| 字段值2 ||| ... ||| 字段值N
```

- 多个去重字段的值用 `|||` 连接
- 字段值会进行标准化处理（trim、JSON stringify 等）

### 4.4 去重范围

- **目标表已有记录**：合并开始时加载的目标表所有记录
- **已合并记录**：当前合并过程中已写入的记录
- **源表内去重**：同一源表内的重复记录也会被处理

## 五、字段拆分功能

### 5.1 配置项

| 配置项 | 说明 |
|--------|------|
| 启用拆分 | 是否开启拆分功能 |
| 主字段 | 值按分隔符拆分的字段 |
| 同步拆分字段 | 与主字段同步拆分的字段（可多选） |
| 分隔符 | 用于拆分字段值的分隔符，如 `|`、`,` 等 |

### 5.2 拆分逻辑

```
原记录:
  主字段: "a1|a2|a3"
  同步字段: "b1|b2|b3"
  其他字段: "xxx"

    ↓ 拆分

拆分后记录 1:
  主字段: "a1"
  同步字段: "b1"
  其他字段: "xxx"

拆分后记录 2:
  主字段: "a2"
  同步字段: "b2"
  其他字段: "xxx"

拆分后记录 3:
  主字段: "a3"
  同步字段: "b3"
  其他字段: "xxx"
```

### 5.3 拆分规则

1. **主字段必须包含分隔符**：否则不拆分，保持原记录
2. **同步字段一一对应**：同步字段的第 N 部分对应主字段的第 N 部分
3. **部分数不足时补空字符串**：如果同步字段部分数少于主字段，不足部分填空字符串
4. **其他字段复制**：非拆分字段的值复制到所有拆分后的记录
5. **富文本支持**：自动识别富文本格式 `[{"type":"text","text":"xxx"}]`，提取纯文本后拆分

### 5.4 拆分时机

```
源记录 → 去重 → 字段映射 → 拆分 → 写入目标表
```

拆分发生在**写入前**，去重判断的是**拆分前的记录**。

### 5.5 父子记录拆分

如果原记录有父记录关联，拆分后的所有记录都继承同一个父记录：

```
原记录（父记录 ID: P001）:
  主字段: "a1|a2"

    ↓ 拆分

拆分后记录 1:
  主字段: "a1"
  父记录: P001

拆分后记录 2:
  主字段: "a2"
  父记录: P001
```

## 六、父子记录关联

### 6.1 自动识别

系统会自动识别源表和目标表中的自关联字段（如"父记录"字段），用于保持父子关系。

### 6.2 写入顺序

```
1. 先写入所有父记录（无 parentRecordId 的记录）
2. 建立源记录 ID → 目标记录 ID 的映射
3. 再写入所有子记录，关联字段使用映射后的目标记录 ID
```

### 6.3 关联字段格式

飞书多维表格关联字段格式：

```json
{
  "type": "text",
  "text": "",
  "recordIds": ["recXXXXXX"],
  "tableId": "tblXXXXXX"
}
```

## 七、完整合并流程

```
┌─────────────────────────────────────────────────────────────┐
│                      合并开始                                │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 1. 加载目标表已有记录（用于去重）                            │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. 遍历每个源表：                                            │
│    ├─ 加载源表记录                                           │
│    ├─ 检测父子关系                                           │
│    ├─ 去重计算                                               │
│    │   ├─ 计算每条记录的去重 Key                              │
│    │   ├─ 检查是否与目标表/已合并记录重复                     │
│    │   └─ 根据策略决定：跳过 / 覆盖（删除目标记录）           │
│    ├─ 字段映射                                               │
│    ├─ 字段拆分（如果启用）                                    │
│    │   ├─ 提取主字段纯文本                                    │
│    │   ├─ 按分隔符拆分                                        │
│    │   └─ 同步拆分其他字段                                    │
│    └─ 写入目标表                                              │
│        ├─ 先写入父记录                                        │
│        └─ 再写入子记录（带关联）                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      合并完成                                │
└─────────────────────────────────────────────────────────────┘
```

**预期结果**：源表 A 的所有记录合并到目标表 B

## 九、注意事项

1. **字段类型兼容**：确保源字段和目标字段类型兼容，否则可能写入失败
2. **分隔符选择**：拆分分隔符应选择不会出现在字段值正常内容中的字符
3. **去重字段选择**：去重字段应选择能唯一标识记录的字段，如索引、ID 等
4. **大量数据处理**：合并大量数据时，系统会分批处理（每批 500 条），请耐心等待
5. **覆盖模式风险**：覆盖模式会删除目标表中的记录，请确认后再使用

## 十、错误排查

### 10.1 常见错误

| 错误信息 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 字段映射失败 | 源字段不存在 | 检查源表字段是否被删除 |
| 写入失败 | 字段类型不兼容 | 检查字段类型是否匹配 |
| 关联字段失败 | 父记录未找到 | 检查父子记录是否在同一批次 |
| 拆分未生效 | 字段值不含分隔符 | 检查分隔符配置是否正确 |

### 10.2 日志关键字

| 关键字 | 说明 |
|--------|------|
| `[去重]` | 去重相关日志 |
| `[拆分]` | 拆分相关日志 |
| `[覆盖]` | 覆盖模式删除记录日志 |
| `[Hierarchy]` | 父子记录处理日志 |
